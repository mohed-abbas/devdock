# Phase 2 — UI Design Contract

> Visual and interaction contract for the authentication phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 2 delivers two screens:** a login page and a placeholder dashboard. The login page is the first user-facing UI in DevDock. The dashboard placeholder establishes the layout shell that Phase 6 builds on. Both must feel polished -- not like scaffolding.

---

## Spacing Scale

Inherited from Phase 1. No changes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, input padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, form field gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, page margins on desktop |
| 3xl | 64px | Page-level vertical spacing |

Exceptions: All interactive elements (buttons, inputs) must have a minimum touch target of 44px height. The shadcn Button `lg` size (h-9 = 36px) is too small for mobile; use `className="h-11"` (44px) for the primary Sign In button. Input fields are already 40px+ by default in base-nova.

**Phase 2 specific spacing:**

| Element | Spacing | Tailwind |
|---------|---------|----------|
| Login card internal padding | 24px (lg) | `p-6` (via Card default px-4, override as needed) |
| Form field vertical gap | 16px (md) | `space-y-4` |
| Title to card gap | 32px (xl) | `mb-8` |
| Error message top margin | 8px (sm) | `mt-2` |
| Dashboard header height | 64px (3xl) | `h-16` |
| Dashboard header horizontal padding | 24px (lg) | `px-6` |

## Color

Inherited from Phase 1. All values use the OKLCH notation from the actual `globals.css` dark theme (DevDock defaults to dark mode).

| Role | Value (dark mode) | Usage |
|------|-------------------|-------|
| Dominant (60%) | `oklch(0.145 0 0)` (--background) | Page background on login and dashboard |
| Secondary (30%) | `oklch(0.205 0 0)` (--card) | Login card surface, dashboard header, sidebar (future) |
| Accent (10%) | `oklch(0.922 0 0)` (--primary) | Sign In button background, focus rings |
| Destructive | `oklch(0.704 0.191 22.216)` (--destructive) | Error message text, rate limit warning |
| Muted foreground | `oklch(0.708 0 0)` (--muted-foreground) | Subtitle text, placeholder text, secondary labels |
| Foreground | `oklch(0.985 0 0)` (--foreground) | Primary text, headings |
| Border | `oklch(1 0 0 / 10%)` (--border) | Card ring, input borders |
| Input | `oklch(1 0 0 / 15%)` (--input) | Input field background tint |

**Accent reserved for (this phase):**
1. Sign In button (primary action, `bg-primary`)
2. Focus rings on form inputs and buttons (`focus-visible:ring-ring`)
3. No other accent usage in this phase

**Destructive reserved for:**
1. Login error message text ("Invalid username or password.")
2. Rate limit warning text ("Too many attempts. Try again in X seconds.")

**Source:** `src/app/globals.css` lines 86-118 (`.dark` block).

## Login Page Layout

### Structure

```
Full viewport, centered both axes
|
+-- Container (max-w-sm = 384px, mx-auto, px-4 for mobile)
    |
    +-- Title block (text-center, mb-8)
    |   +-- "DevDock" (display type, foreground color)
    |   +-- "Remote development platform" (body type, muted-foreground)
    |
    +-- Card (shadcn Card component)
        |
        +-- CardContent (pt-6)
            |
            +-- Form (space-y-4)
                |
                +-- Field group (space-y-2)
                |   +-- Label "Username"
                |   +-- Input (type="text", required, autoComplete="username")
                |
                +-- Field group (space-y-2)
                |   +-- Label "Password"
                |   +-- Input (type="password", required, autoComplete="current-password")
                |
                +-- Error region (aria-live="polite", min-h-5)
                |   +-- Error text (if error) OR empty (if no error)
                |
                +-- Button "Sign In" (full width, h-11 for 44px touch target)
```

### Specifications

| Property | Value |
|----------|-------|
| Page background | bg-background (full viewport) |
| Container width | max-w-sm (384px) with px-4 mobile padding |
| Vertical centering | min-h-svh flex items-center justify-center |
| Title "DevDock" | text-2xl font-semibold tracking-tight text-foreground |
| Subtitle | text-base text-muted-foreground mt-2 |
| Card | shadcn Card (no CardHeader, use CardContent only for cleaner layout) |
| Card top padding | pt-6 on CardContent |
| Card bottom padding | pb-6 on CardContent (default from Card) |
| Form field gap | space-y-4 (16px) |
| Label-to-input gap | space-y-2 (8px) within each field group |
| Error area | Fixed height (min-h-[20px]) to prevent layout shift. Uses aria-live="polite" for screen readers. |
| Error text | text-sm text-destructive |
| Button | Full width (w-full), h-11 (44px), default variant (bg-primary) |
| Button label (default) | "Sign In" |
| Button label (loading) | "Signing in..." with disabled state |
| Autofocus | Username input gets autoFocus on page load |

### No elements present

These elements are explicitly NOT on the login page (per CONTEXT.md decisions):
- No logo or icon (D-02)
- No illustration or decorative image (D-02)
- No "Remember me" checkbox (D-03)
- No "Forgot password" link (single admin tool, not a SaaS)
- No "Sign up" link (users created via CLI seed script)
- No social login buttons (GitHub OAuth is for repo access, not login)

## Interaction States

### Login Form States

| State | Visual Treatment |
|-------|-----------------|
| **Default** | Empty form, username input autofocused. Error area empty (min-height preserved). Sign In button enabled. |
| **Typing** | Input value visible. No validation until submit. |
| **Submitting** | Button shows "Signing in..." and is disabled (`disabled` attribute). Inputs remain visible but form is non-interactive (form action handles this). No spinner -- text change is sufficient for a sub-second operation. |
| **Error (invalid credentials)** | Error area shows "Invalid username or password." in destructive color. Form fields retain their values. Focus moves to username input. Button re-enables. |
| **Error (rate limited)** | Error area shows "Too many attempts. Try again in 30 seconds." in destructive color. Button remains disabled for the duration. No countdown timer -- the message is static. User retries manually after waiting. |
| **Success** | Redirect to /dashboard. No success message shown on login page (user sees the dashboard immediately). |

### Input Focus States

| State | Visual |
|-------|--------|
| Default | border-input background, no ring |
| Focused | border-ring, ring-3 ring-ring/50 (shadcn default focus style) |
| Invalid (after submit error) | No special invalid styling on individual inputs -- error is form-level per D-04 |

### Sign Out Interaction

| State | Visual |
|-------|--------|
| Default | Ghost button, muted text |
| Hover | bg-muted text-foreground (ghost hover) |
| Clicked | Immediate redirect to /login. No confirmation dialog (non-destructive action). |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Sign In" |
| Primary CTA (loading) | "Signing in..." |
| Login title | "DevDock" |
| Login subtitle | "Remote development platform" |
| Username label | "Username" |
| Username placeholder | (none -- leave empty, label is sufficient) |
| Password label | "Password" |
| Password placeholder | (none -- leave empty, label is sufficient) |
| Error: invalid credentials | "Invalid username or password." |
| Error: rate limited | "Too many attempts. Try again in 30 seconds." |
| Error: server error | "Something went wrong. Please try again." |
| Dashboard welcome | "Welcome, {username}" |
| Dashboard header brand | "DevDock" |
| Sign out button | "Sign Out" |
| Empty state heading | Not applicable (dashboard placeholder has no data views) |
| Empty state body | Not applicable |
| Destructive confirmation | Not applicable (no destructive actions in this phase) |

**Copywriting rules:**
- No exclamation marks. Tone is calm and professional.
- Error messages use period at end. They describe what happened, not what the user did wrong.
- CTA uses "Sign In" (two words, title case), not "Login" or "Log In" as a verb.
- "Sign Out" (two words, title case), not "Logout" or "Log Out".

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Form labels | Every input has a `<Label>` with matching `htmlFor`/`id` |
| Error announcement | Error region uses `aria-live="polite"` so screen readers announce errors |
| Focus management | Username input has `autoFocus` on page load. After error, focus returns to username input. |
| Touch targets | Sign In button is 44px tall (h-11). Input fields are 40px+ (shadcn default). |
| Keyboard navigation | Tab order: username -> password -> sign in button. Enter submits the form. |
| Autocomplete | Username input: `autoComplete="username"`. Password input: `autoComplete="current-password"`. |
| Color contrast | Destructive text on dark background exceeds 4.5:1 ratio. Muted foreground on dark background exceeds 4.5:1. |

## Responsive Behavior

| Breakpoint | Login Page | Dashboard |
|------------|------------|-----------|
| Mobile (<640px) | Card stretches to full width minus 16px padding each side. Same vertical centering. | Header collapses to logo + sign out (no username text). Welcome centered. |
| Tablet (640-1024px) | Card at max-w-sm (384px), centered. | Full header with username. Welcome centered. |
| Desktop (>1024px) | Same as tablet. Login card does not grow wider. | Full header. Welcome centered. Phase 6 will add sidebar and content grid. |

**Mobile-specific:**
- Login card: `w-full max-w-sm` ensures it fills narrow viewports without overflowing.
- Dashboard header: On screens below 640px, hide the username text (show only "DevDock" and "Sign Out"). Use `hidden sm:block` on the username span.
