# Phase 4 — UI Design Contract

> Visual and interaction contract for the web terminal phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 4 delivers:** a dedicated full-screen terminal page (`/dashboard/env/[id]/terminal`), tabbed multi-terminal sessions via xterm.js + Socket.IO, auto-reconnect overlay, and a "Terminal" button on existing environment cards. No dashboard polish (Phase 6), no resource limits (Phase 7), no GitHub integration (Phase 5).

---

## Terminal Page Layout

### Route

`/dashboard/env/[id]/terminal` -- dedicated page, navigated from environment card.

### Structure (D-01, D-02, D-12)

```
Terminal page (full viewport height, flex column)
|
+-- Header bar (h-12, bg-card, border-b, sticky top-0, z-50)
|   +-- Left: Back button (ChevronLeft icon, variant="ghost" size="sm")
|   +-- Left: Environment name (text-sm font-semibold, truncated)
|   +-- Left: Status indicator (dot, same colors as StatusBadge)
|   +-- Right: (reserved for future controls)
|
+-- Tab bar (h-9, bg-card, border-b, flex items-center px-2)
|   +-- Tab item (repeated, inline-flex items-center px-3 gap-1)
|   |   +-- Tab label ("Terminal 1", "Terminal 2", ...)
|   |   +-- Close button (X icon, size-3, visible on hover/active)
|   +-- New tab button (Plus icon, size-9, variant="ghost")
|
+-- Terminal container (flex-1, relative, overflow-hidden)
    +-- xterm.js canvas (fills container via addon-fit)
    +-- Reconnect overlay (absolute, centered, shown during disconnect)
```

### Specifications

| Property | Value |
|----------|-------|
| Page layout | `flex flex-col h-svh` (full viewport, no scroll) |
| Header height | `h-12` (48px) |
| Header background | `bg-card` |
| Header border | `border-b border-border` |
| Header z-index | `z-50` |
| Back button | `variant="ghost" size="sm"`, ChevronLeft icon, navigates to `/dashboard` |
| Environment name | `text-sm font-semibold text-foreground truncate max-w-[200px]` |
| Status dot in header | `size-2 rounded-full` with status color (same mapping as StatusBadge) |
| Tab bar height | `h-9` (36px) |
| Tab bar background | `bg-card` (same as header, continuous surface) |
| Tab bar border | `border-b border-border` |
| Active tab | `text-foreground border-b-2 border-primary` |
| Inactive tab | `text-muted-foreground hover:text-foreground` |
| Tab close button | `size-3 text-muted-foreground hover:text-destructive` hidden until tab hover |
| New tab button | `variant="ghost" size="sm"` with Plus icon, `size-4` |
| Terminal container | `flex-1 relative overflow-hidden bg-background p-0` |
| xterm.js font family | `var(--font-geist-mono), monospace` |
| xterm.js font size | `14px` |
| xterm.js line height | `1.2` |
| xterm.js scroll behavior | Scrollback buffer 5000 lines |

### Tab Lifecycle (Claude's discretion)

| Property | Value |
|----------|-------|
| Max tabs | 5 per environment |
| Default tab | 1 tab auto-opens on page load labeled "Terminal 1" |
| Tab naming | "Terminal 1", "Terminal 2", ..., "Terminal 5" (sequential) |
| Close last tab | Navigates back to `/dashboard` |
| Tab exceeded message | New tab button disabled, tooltip "Maximum 5 terminals" |
| Tab persistence | Tabs do not persist across page navigation -- each visit starts fresh |

## Reconnect Overlay (D-06)

### Structure

```
Overlay (absolute inset-0, bg-background/80 backdrop-blur-sm, flex center)
|
+-- Content card (bg-card, rounded-lg, p-6, max-w-sm, text-center)
    +-- Loader icon (RefreshCw, animate-spin, size-8, muted-foreground)
    +-- "Reconnecting..." heading (mt-4)
    +-- "Attempting to restore terminal session." body (mt-2)
```

After ~30 seconds of failed reconnection:

```
Overlay (same container)
|
+-- Content card (same styling)
    +-- AlertCircle icon (size-8, destructive)
    +-- "Connection lost" heading (mt-4, text-destructive)
    +-- "The terminal session could not be restored." body (mt-2)
    +-- "Retry" button (mt-4, default variant)
    +-- "Back to Dashboard" link (mt-2, text-sm, muted-foreground, underline)
```

### Specifications

| Property | Value |
|----------|-------|
| Overlay background | `bg-background/80 backdrop-blur-sm` |
| Overlay position | `absolute inset-0 z-40 flex items-center justify-center` |
| Content card | `bg-card rounded-lg p-6 max-w-sm text-center shadow-lg` |
| Reconnecting icon | `RefreshCw` from lucide, `size-8 text-muted-foreground animate-spin` |
| Reconnecting heading | "Reconnecting..." -- `text-xl font-semibold text-foreground mt-4` |
| Reconnecting body | "Attempting to restore terminal session." -- `text-base text-muted-foreground mt-2` |
| Connection lost icon | `AlertCircle` from lucide, `size-8 text-destructive` |
| Connection lost heading | "Connection lost" -- `text-xl font-semibold text-destructive mt-4` |
| Connection lost body | "The terminal session could not be restored." -- `text-base text-muted-foreground mt-2` |
| Retry button | "Retry" -- `variant="default" mt-4` |
| Dashboard link | "Back to Dashboard" -- `text-sm text-muted-foreground underline mt-2` |
| Reconnect timeout | 30 seconds before switching to "Connection lost" state |

## Interaction States

### Terminal Page States

| State | Visual Treatment |
|-------|-----------------|
| **Loading** | Terminal container shows skeleton shimmer. Header shows env name and status dot. |
| **Connected** | xterm.js canvas visible and interactive. Cursor blinking. Tab active. |
| **Reconnecting** | Reconnect overlay with spinning RefreshCw. Terminal dimmed behind overlay. |
| **Connection lost** | Connection lost overlay with AlertCircle. Retry button and dashboard link. |
| **Tab switching** | Active tab highlighted with primary border. Previous tab content hidden. New tab content shown. Instant switch (no animation). |
| **New tab opening** | New Docker exec session created. "Terminal N" label appears in tab bar. Focus moves to new tab. |
| **Closing tab** | Tab removed. Adjacent tab becomes active. If last tab, navigate to dashboard. |
| **Environment not running** | Redirect to dashboard with error state. Terminal page requires running environment. |

### Tab States

| State | Visual Treatment |
|-------|-----------------|
| **Active** | `text-foreground border-b-2 border-primary` |
| **Inactive** | `text-muted-foreground` |
| **Hover (inactive)** | `text-foreground`, close button becomes visible |
| **Close hover** | Close button `text-destructive` |

### Terminal Loading State

| Scenario | Visual |
|----------|--------|
| **Initial page load** | Skeleton placeholder matching terminal area. Header and tab bar render immediately. |
| **WebSocket connecting** | Brief loading state, then terminal appears. |
| **New tab creating** | Tab appears immediately, terminal connects within ~500ms. |

- Terminal renders and accepts touch keyboard input
- Tab labels truncate with ellipsis if needed: `truncate max-w-[80px]`

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Tab keyboard navigation | Arrow keys switch between tabs. Tab key moves to terminal content. |
| Tab ARIA roles | Tab bar uses `role="tablist"`, each tab uses `role="tab"`, terminal panel uses `role="tabpanel"` |
| Active tab state | `aria-selected="true"` on active tab |
| Terminal focus | xterm.js canvas receives focus when tab is activated |
| Close tab keyboard | Tab close button accessible via Tab key within the tab |
| Reconnect overlay | `aria-live="assertive"` on overlay content so screen readers announce connection state |
| Back button | Proper `aria-label="Back to dashboard"` |
| New tab button | `aria-label="New terminal"` or `aria-label="Maximum 5 terminals"` when disabled |
| Terminal button on card | `aria-label="Open terminal"` |
| Touch targets | All header/tab buttons minimum 36px (tab items), action buttons 44px |

## xterm.js Addon Configuration

| Addon | Import | Purpose |
|-------|--------|---------|
| `@xterm/addon-fit` | `FitAddon` | Auto-resize terminal to container on window resize |
| `@xterm/addon-web-links` | `WebLinksAddon` | Make URLs clickable in terminal output |
| `@xterm/addon-search` | `SearchAddon` | Ctrl+Shift+F to search scrollback buffer |
| `@xterm/addon-clipboard` | `ClipboardAddon` | Cross-browser copy/paste support |

**Fit behavior:**
- Call `fitAddon.fit()` on mount, on window resize, on tab switch
- Use `ResizeObserver` on the terminal container div for accurate sizing
- Send new dimensions (cols, rows) to server via Socket.IO `resize` event
