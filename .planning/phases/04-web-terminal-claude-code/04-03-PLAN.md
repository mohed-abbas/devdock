**Step 2: Create `src/app/dashboard/env/[id]/terminal/page.tsx`** -- Server component:

```typescript
import { redirect } from 'next/navigation';
import { auth } from '@/auth';
import { db } from '@/lib/db';
import { environments } from '@/lib/db/schema';
import { eq, and } from 'drizzle-orm';
import dynamic from 'next/dynamic';

// Dynamic import with SSR disabled (Pitfall 6: xterm.js requires browser APIs)
const TerminalClient = dynamic(
  () => import('./_components/terminal-client').then(mod => ({ default: mod.TerminalClient })),
  { ssr: false, loading: () => <TerminalSkeleton /> }
);

function TerminalSkeleton() {
  return (
    <div className="flex flex-col h-svh">
      <div className="h-12 bg-card border-b border-border" />
      <div className="h-8 bg-card border-b border-border" />
      <div className="flex-1 bg-background" />
    </div>
  );
}

type Params = { params: Promise<{ id: string }> };

export default async function TerminalPage({ params }: Params) {
  const { id } = await params;
  const session = await auth();
  if (!session?.user?.id) redirect('/login');

  const [env] = await db
    .select({
      id: environments.id,
      name: environments.name,
      slug: environments.slug,
      status: environments.status,
    })
    .from(environments)
    .where(and(
      eq(environments.id, id),
      eq(environments.userId, session.user.id),
    ))
    .limit(1);

  if (!env) redirect('/dashboard');
  if (env.status !== 'running') redirect('/dashboard');

  return (
    <TerminalClient
      environmentId={env.id}
      environmentName={env.name}
      environmentStatus={env.status}
    />
  );
}
```

**Step 6: Create `src/app/dashboard/env/[id]/terminal/_components/terminal-client.tsx`:**

`'use client'` -- the main orchestration component. This is the largest and most complex component.

Props: `{ environmentId: string; environmentName: string; environmentStatus: string }`

State management (use React useState, not zustand -- simpler for this scope):
- `tabs`: array of `{ id: number; label: string; terminalRef: React.RefObject<Terminal | null> }`
- `activeTabId`: number
- `connectionState`: `'connecting' | 'connected' | 'reconnecting' | 'disconnected'`
- `nextTabId`: counter starting at 1
- `socketRef`: `useRef<Socket | null>(null)`

**Connection flow:**
1. On mount, fetch token from `POST /api/terminal/token` with `{ environmentId }`
2. Connect Socket.IO: `io('/terminal', { path: '/ws/socket.io', auth: { token }, reconnection: true, reconnectionAttempts: 10, reconnectionDelay: 1000, reconnectionDelayMax: 5000 })`
3. On `connect`: set state `'connected'`, create first tab
4. On `disconnect`: set state `'reconnecting'`
5. On `reconnect`: set state `'connected'`
6. On `reconnect_failed`: set state `'disconnected'` (after ~30s per Socket.IO attempts)
7. On `connect_error`: if first connection, show error

**Tab management:**
- `addTab()`: increment nextTabId, add `{ id: nextTabId, label: "Terminal {N}" }`, set active, emit `exec:create` with terminal dimensions
- `closeTab(tabId)`: remove tab, if was active switch to adjacent, if no tabs left navigate to `/dashboard`
- `switchTab(tabId)`: set activeTabId, the TerminalInstance for the new tab gets `visible=true`
- Max 5 tabs (D-02, UI-SPEC)

**Socket event handlers:**
- `exec:created`: tab is ready, focus terminal
- `exec:output`: write data to the corresponding tab's terminal: `tab.terminalRef.current?.write(data)`
- `exec:exit`: close the tab
- `exec:error`: show error in terminal: `tab.terminalRef.current?.write('\r\n\x1b[31m' + message + '\x1b[0m\r\n')`

**Data handlers:**
- `onData(tabId, data)`: emit `exec:input` with `{ sessionIndex: tabIndex, data }`
- `onResize(tabId, cols, rows)`: emit `exec:resize` with `{ sessionIndex: tabIndex, cols, rows }`

**Retry logic:**
- `onRetry()`: disconnect old socket, fetch new token, create new connection

**Layout (per UI-SPEC Terminal Page Layout):**
```
<div className="flex flex-col h-svh">

  </div>

  {/* Tab bar */}
  <TerminalTabs
    tabs={tabs}
    activeTab={activeTabId}
    onTabSwitch={switchTab}
    onTabClose={closeTab}
    onNewTab={addTab}
    maxTabs={5}
  />

  {/* Terminal container */}
  <div className="flex-1 relative overflow-hidden bg-background p-0">
    {tabs.map((tab, index) => (
      <TerminalInstance
        key={tab.id}
        visible={tab.id === activeTabId}
        onData={(data) => onData(tab.id, data)}
        onResize={(cols, rows) => onResize(tab.id, cols, rows)}

      />
    ))}

    />

```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Terminal page | Auth.js middleware protects /dashboard/* routes |
| Browser -> Token API | Session cookie validates user before token generation |
| Browser -> Socket.IO | Signed token in auth parameter validated before upgrade |
| xterm.js -> Socket.IO | User input flows to exec stdin only in their own container |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-14 | Spoofing | Terminal page access | mitigate | Server component validates session + environment ownership before rendering. Non-running envs redirect to dashboard. |
| T-04-15 | Information Disclosure | xterm.js SSR crash | mitigate | Dynamic import with `ssr: false` prevents server-side import of browser-only xterm.js (Pitfall 6). |
| T-04-16 | Denial of Service | Tab bomb | mitigate | Max 5 tabs enforced in UI (new tab button disabled at limit). Server-side sessions cleaned up on disconnect. |
| T-04-17 | Tampering | sessionIndex manipulation | accept | Client controls sessionIndex for its own exec sessions. Worst case: user writes to their own wrong terminal tab. No cross-user impact. |
</threat_model>
