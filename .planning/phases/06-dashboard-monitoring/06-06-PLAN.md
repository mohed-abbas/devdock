**A) Create `deploy/nginx/devdock-preview.conf`** -- A new nginx server block for wildcard subdomain preview routing.

Architecture: nginx receives `*.preview.devdock.example.com` requests and proxies them to the Next.js app on port 3000. Next.js extracts the subdomain, looks up the environment, finds the container IP via Docker inspect, and proxies to the container. This eliminates HTML rewriting entirely because the browser sees the app at the root of its own subdomain.

The server_name placeholder uses `*.preview.devdock.yourdomain.com` matching the existing devdock.conf pattern. The user will update the domain when deploying.

```nginx
# deploy/nginx/devdock-preview.conf
# Wildcard subdomain proxy for DevDock environment previews (D-17)
# Each environment is accessed via {env-id}.preview.devdock.yourdomain.com
#
# Prerequisites:
#   1. Wildcard DNS: *.preview.devdock.yourdomain.com -> VPS IP
#   2. Wildcard TLS cert (certbot --manual --preferred-challenges dns -d "*.preview.devdock.yourdomain.com")
#   3. Symlink: sudo ln -s /path/to/devdock-preview.conf /etc/nginx/sites-enabled/
#
# Install: sudo ln -s /home/murx-dev/devdock/deploy/nginx/devdock-preview.conf /etc/nginx/sites-enabled/devdock-preview.conf

map $http_upgrade $preview_connection_upgrade {
    default upgrade;
    '' close;
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name *.preview.devdock.yourdomain.com;

    # Wildcard TLS cert -- update paths after certbot setup
    ssl_certificate /etc/letsencrypt/live/preview.devdock.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/preview.devdock.yourdomain.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers off;

    # Security headers
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # Proxy all requests to Next.js which handles subdomain extraction + container routing
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;

        # WebSocket support (for HMR in previewed apps)
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $preview_connection_upgrade;

        # Pass the original Host so Next.js can extract the subdomain
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_buffering off;
        proxy_cache off;

        # Generous timeout for dev apps that may be slow
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }
}
```

**C) Create `src/app/api/preview/[[...path]]/route.ts`** -- The subdomain-aware preview proxy route.

This route handles ALL requests coming through the wildcard subdomain nginx block. It:
1. Extracts the environment ID from the subdomain (first segment of Host header before `.preview.`)
2. Looks up the environment in the DB (no auth check needed -- the environment ID in the subdomain acts as an unguessable token since IDs are UUIDs)
3. Finds the container's Docker network IP via `docker inspect`
4. Proxies the request to `http://{containerIp}:{previewPort}{path}`

Key implementation details:
- Extract subdomain from `request.headers.get('host')` -- split on the first `.` to get the env ID prefix, then validate it matches a UUID pattern
- Use the same Docker inspect pattern as the existing proxy at `src/app/api/environments/[id]/preview/[[...path]]/route.ts` (lines 37-58) for container IP lookup
- Strip sensitive headers (`cookie`, `authorization`) before forwarding (T-06-02 from existing proxy)
- Set `host` header to `{containerIp}:{previewPort}` on forwarded request
- Do NOT rewrite HTML -- the whole point of subdomain proxying is that paths resolve naturally
- Support all HTTP methods (GET, POST, PUT, PATCH, DELETE) via separate named exports
- On upstream fetch error, return 502 with message "Upstream connection failed"
- If environment not found or not running or no previewPort, return 404 "Preview not available"
- If container not found or IP unavailable, return 503

The route file structure:
```typescript
import { NextRequest, NextResponse } from 'next/server';
import { db } from '@/lib/db';
import { environments } from '@/lib/db/schema';
import { eq } from 'drizzle-orm';
import Docker from 'dockerode';
import { config } from '@/lib/config';

const docker = new Docker({ socketPath: config.DOCKER_SOCKET });

const STRIP_HEADERS = ['cookie', 'authorization'];

function extractEnvIdFromHost(host: string | null): string | null {
  if (!host) return null;
  // Host looks like: {env-id}.preview.devdock.example.com
  // Extract everything before the first dot that matches PREVIEW_DOMAIN
  const previewDomain = config.PREVIEW_DOMAIN;
  if (!previewDomain || !host.endsWith(previewDomain)) return null;
  const prefix = host.slice(0, host.length - previewDomain.length - 1); // -1 for the dot
  // Validate UUID format (basic check)
  if (!/^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i.test(prefix)) {
    return null;
  }
  return prefix;
}

async function proxyRequest(request: NextRequest) {
  const host = request.headers.get('host');
  const envId = extractEnvIdFromHost(host);
  if (!envId) {
    return new NextResponse('Preview not available', { status: 404 });
  }

  // Look up environment -- no user auth needed, UUID is unguessable
  const [env] = await db
    .select()
    .from(environments)
    .where(eq(environments.id, envId))
    .limit(1);

  if (!env || env.status !== 'running' || !env.previewPort) {
    return new NextResponse('Preview not available', { status: 404 });
  }

  // Find dev container by compose labels
  const containers = await docker.listContainers({
    filters: {
      label: [
        `com.docker.compose.project=${env.dockerProjectName}`,
        'com.docker.compose.service=dev',
      ],
    },
  });

  if (containers.length === 0) {
    return new NextResponse('Container not found', { status: 503 });
  }

  const inspection = await docker.getContainer(containers[0].Id).inspect();
  const networks = inspection.NetworkSettings.Networks;
  const networkName = Object.keys(networks)[0];
  const containerIp = networks[networkName]?.IPAddress;

  if (!containerIp) {
    return new NextResponse('Container IP unavailable', { status: 503 });
  }

  // Build target URL -- path is relative to subdomain root (no rewriting needed!)
  const url = new URL(request.url);
  const targetUrl = `http://${containerIp}:${env.previewPort}${url.pathname}${url.search}`;

  const forwardHeaders = new Headers(request.headers);
  for (const header of STRIP_HEADERS) {
    forwardHeaders.delete(header);
  }
  forwardHeaders.set('host', `${containerIp}:${env.previewPort}`);

  try {
    const upstreamRes = await fetch(targetUrl, {
      method: request.method,
      headers: forwardHeaders,
      body: request.method !== 'GET' && request.method !== 'HEAD' ? request.body : undefined,
      // @ts-expect-error -- duplex needed for streaming body
      duplex: 'half',
    });

    const responseHeaders = new Headers(upstreamRes.headers);
    responseHeaders.delete('transfer-encoding');
    responseHeaders.delete('content-encoding');

    return new NextResponse(upstreamRes.body, {
      status: upstreamRes.status,
      headers: responseHeaders,
    });
  } catch {
    return new NextResponse('Upstream connection failed', { status: 502 });
  }
}

export const GET = proxyRequest;
export const POST = proxyRequest;
export const PUT = proxyRequest;
export const PATCH = proxyRequest;
export const DELETE = proxyRequest;
```

IMPORTANT: This route has NO `auth()` call. Authentication is intentionally skipped because:
- Environment IDs are UUIDs (unguessable)
- The previewed app may make its own API calls that shouldn't need DevDock cookies
- Adding auth would break the previewed app's own asset/API requests
- This matches the pattern of services like Vercel Preview URLs which are accessible by URL

The `PREVIEW_DOMAIN` config check in `extractEnvIdFromHost` ensures this route only activates when the preview subdomain feature is configured.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet -> nginx -> Next.js | Preview subdomain requests cross from public internet through nginx to Next.js |
| Next.js -> Container | Proxied requests cross from DevDock host into user container Docker network |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06-01 | Information Disclosure | Preview proxy | mitigate | Strip `cookie` and `authorization` headers before forwarding to container -- prevents DevDock session cookie leaking into user app process |
| T-06-06-02 | Spoofing | Preview proxy | accept | No auth on preview URLs -- environment UUIDs are unguessable (122 bits entropy). Acceptable for dev tool preview functionality. Same model as Vercel/Netlify preview URLs. |
| T-06-06-03 | Tampering | Nginx config | mitigate | Nginx config is a static file checked into repo -- not generated dynamically. No injection vector from user input. |
| T-06-06-04 | Elevation of Privilege | Preview proxy | mitigate | Proxy only forwards HTTP requests to container's preview port -- no exec, no Docker socket access, no write to DB. Read-only proxy. |
| T-06-06-05 | Denial of Service | Preview proxy | accept | Container apps may be slow or unresponsive -- 300s timeout in nginx config. No rate limiting on preview (dev tool, not public-facing production). |
</threat_model>
