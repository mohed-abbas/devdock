   This must happen at the top of the connection handler, before the Docker logs setup.

2. **Forward exec output to the container room.** In the `terminalNs.on('connection')` handler, inside the `stream.on('data')` callback (line ~82), after the existing `socket.emit('exec:output', ...)` call, add a broadcast to the /logs namespace room:
   ```
   logsNs.to(`container:${containerId}`).emit('logs:data', {
     data: chunk.toString('utf-8'),
   });
   ```
   The `containerId` variable is already available in scope from `socket.data` destructuring at line 62.

3. **Keep the existing Docker container.logs() streaming in /logs.** Do NOT remove or disable the existing container.logs() streaming. The exec forwarding supplements it with terminal session output.
