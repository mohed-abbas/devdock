# Phase 3 — UI Design Contract

> Visual and interaction contract for the environment lifecycle phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 3 delivers three UI areas:** a dashboard environment list with status badges, a creation dialog (modal), and a delete confirmation dialog. The dashboard placeholder from Phase 2 is replaced with a real environment list that polls for status updates every 3 seconds. No web terminal (Phase 4), no GitHub browse-and-select (Phase 5), no dashboard polish (Phase 6).

## Color

Inherited from Phase 1/2. All values use OKLCH notation from `globals.css` dark theme.

| Role | Value (dark mode) | Usage |
|------|-------------------|-------|
| Dominant (60%) | `oklch(0.145 0 0)` (--background) | Page background |
| Secondary (30%) | `oklch(0.205 0 0)` (--card) | Environment cards, dialog surfaces, header |
| Accent (10%) | `oklch(0.922 0 0)` (--primary) | "New Environment" button, "Create & Start" button, focus rings |
| Destructive | `oklch(0.704 0.191 22.216)` (--destructive) | Delete button, error status badge, error messages |
| Muted foreground | `oklch(0.708 0 0)` (--muted-foreground) | Repo URLs, timestamps, empty state body, placeholder text |
| Foreground | `oklch(0.985 0 0)` (--foreground) | Card names, headings, primary text |
| Border | `oklch(1 0 0 / 10%)` (--border) | Card ring, separator lines |

**Focal point:** The environment card grid is the primary visual anchor; the "New Environment" button is the secondary focal point in the header.

**Accent reserved for (this phase):**
1. "New Environment" primary CTA button (`bg-primary`)
2. "Create & Start" submit button in creation dialog (`bg-primary`)
3. Focus rings on form inputs, checkboxes, and buttons (`focus-visible:ring-ring`)
4. No other accent usage in this phase

**Status colors (defined in Phase 1, applied here):**

| Status | Color Token | Background | Text | Dot |
|--------|-------------|------------|------|-----|
| running | green | `bg-emerald-500/10` | `text-emerald-400` | `bg-emerald-500` |
| stopped | muted gray | `bg-muted` | `text-muted-foreground` | `bg-muted-foreground` |
| starting | amber | `bg-amber-500/10` | `text-amber-400` | `bg-amber-500` |
| stopping | amber | `bg-amber-500/10` | `text-amber-400` | `bg-amber-500` |
| error | red | `bg-destructive/10` | `text-destructive` | `bg-destructive` |

**Implementation note:** Status colors use Tailwind's built-in color palette (emerald, amber) for the status-specific hues. These do NOT override the shadcn theme variables -- they are applied only to status badge components. The 10% alpha backgrounds ensure badges are readable on the card surface.

### Validation

| Field | Rule | Error message |
|-------|------|---------------|
| Name | Required, 1-100 characters | "Name is required." |
| Name | Must produce a valid slug (letters, numbers, hyphens) | "Name must contain at least one letter or number." |

## Interaction States

### Environment Card States

| State | Visual Treatment |
|-------|-----------------|
| **Running** | Green status badge. Stop button visible. Start button hidden. Delete button visible but secondary. |
| **Stopped** | Gray status badge. Start button visible. Stop button hidden. Delete button visible. |
| **Starting** | Amber badge with pulsing dot. Both Start and Stop buttons disabled. Delete button disabled. |
| **Stopping** | Amber badge with pulsing dot. Both Start and Stop buttons disabled. Delete button disabled. |
| **Error** | Red status badge. Card shows error indicator. Click/hover on badge reveals error message in a tooltip. Start button visible (to retry). Delete button visible. |

### Card Action Buttons

| Button | Visible When | Icon | Variant | Tooltip |
|--------|-------------|------|---------|---------|
| Start | stopped, error | `Play` (lucide) | outline, size="sm" | "Start environment" |
| Stop | running | `Square` (lucide) | outline, size="sm" | "Stop environment" |
| Delete | stopped, running, error | `Trash2` (lucide) | destructive, size="sm" | "Delete environment" |
| (all disabled) | starting, stopping | -- | -- | -- |

### Creation Dialog States

| State | Visual Treatment |
|-------|-----------------|
| **Default (closed)** | "New Environment" button in page header. |
| **Open** | Dialog overlay with form. Name input autofocused. |
| **Filling** | User types name and optional URL, toggles checkboxes. No live validation. |
| **Submitting** | "Create & Start" becomes "Creating..." and is disabled. "Discard" button also disabled. Dialog stays open. |
| **Success** | Dialog closes. New card appears in list with "starting" status (picked up by next poll). |
| **Error** | Error message appears in error region. Form remains editable. Submit button re-enables. |

### Delete Dialog States

| State | Visual Treatment |
|-------|-----------------|
| **Open** | Alert dialog with environment name in title. |
| **Confirming** | "Delete Environment" becomes "Deleting..." and is disabled. "Keep Environment" also disabled. |
| **Success** | Dialog closes. Card removed from list on next poll. |
| **Error** | Dialog closes. Error shown as a toast or inline message on the card (implementation discretion). |

### Loading States

| Scenario | Visual |
|----------|--------|
| **Initial page load** | 3 skeleton cards in the grid. Skeletons match card dimensions with shimmer. |
| **Polling update** | No loading indicator. Data updates silently. Cards animate layout changes with CSS transitions. |
| **No environments** | Empty state immediately (no skeleton). |

## Error State on Card (D-07)

When an environment has status `error`:

| Property | Value |
|----------|-------|
| Badge | Red status badge ("Error") |
| Error detail | Tooltip on hover/click of the status badge showing the error message from DB |
| Tooltip content | `text-sm text-destructive` -- short message like "Container exited with code 1" |
| Tooltip trigger | The status badge itself is the hover/click target |
| Tooltip implementation | Use shadcn Tooltip or a simple title attribute (implementation discretion -- tooltip preferred for mobile) |
| Fallback | If no error message stored, show "An unknown error occurred." |

**Note on schema:** D-07 requires an `errorMessage` column in the `environments` table. The current schema (`src/lib/db/schema.ts`) does not include this field. The planner must add `errorMessage: text('error_message')` to the environments table.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Dialog focus trap | shadcn Dialog and AlertDialog trap focus automatically (Radix/Base UI primitive) |
| Dialog close on Escape | Built into shadcn Dialog and AlertDialog |
| Form labels | Every input and checkbox has a `<Label>` with matching `htmlFor`/`id` |
| Error announcement | Error regions use `aria-live="polite"` so screen readers announce errors |
| Button labels for icon-only buttons | Start/Stop/Delete buttons have tooltip text AND `aria-label` attributes |
| Status announcement | Status badge text provides the status for screen readers; no ARIA needed beyond text content |
| Touch targets | All buttons are minimum 44px (use `h-11` or `size="sm"` + adequate padding). Checkbox click area extends to full label row. |
| Keyboard navigation | Tab navigates between cards (via action buttons). Enter/Space triggers actions. |
| Destructive action guard | Delete requires explicit confirmation via AlertDialog -- cannot be triggered by accident |

## Responsive Behavior

| Breakpoint | Layout | Notes |
|------------|--------|-------|
| Mobile (<768px) | Single column card grid. Full-width cards. Dialog is near-fullscreen (sheet-like). | Action buttons stack if space is tight. |
| Tablet (768-1280px) | 2-column card grid. Dialog at 448px width. | Standard layout. |
| Desktop (>1280px) | 3-column card grid. Dialog at 448px width. | Most spacious layout. |

**Mobile-specific:**
- Card action buttons remain icon-only (no text labels) on all breakpoints for consistency.
- Dialog opens with a backdrop overlay. On mobile, it takes most of the viewport width (`w-[calc(100%-2rem)]`).
- Empty state CTA button is full-width on mobile.
