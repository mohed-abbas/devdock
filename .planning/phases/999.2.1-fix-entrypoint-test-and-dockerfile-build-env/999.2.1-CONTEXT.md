- No change to the entrypoint's other steps (GID fixup, drizzle-kit push, seed, gosu)

### GAP-2 — Dockerfile builder placeholder env vars
- `Dockerfile` builder stage MUST provide harmless build-time placeholders for `DATABASE_URL` and `AUTH_SECRET` via `ARG` + `ENV`, so Next.js page-data collection can evaluate `src/lib/config.ts:loadConfig()` without throwing

- Placeholders MUST NOT leak into the runtime image — only set in the `builder` stage
- Real values flow at runtime via `env_file: .env` in `docker-compose.yml`
