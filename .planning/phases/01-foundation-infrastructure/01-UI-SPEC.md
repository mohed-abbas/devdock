# Phase 1 — UI Design Contract

> Visual and interaction contract for the foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 1 is infrastructure-only.** There are no user-facing features beyond "the app responds behind HTTPS" (CONTEXT.md). This contract establishes the design system foundation that all subsequent phases (2-7) inherit. The focus is on initializing shadcn/ui, locking design tokens, and defining the root layout shell.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (CLI v4.2.0) |
| Preset | default (initialized via `npx shadcn@latest init`) |
| Component library | Radix UI (via shadcn/ui primitives) |
| Icon library | Lucide React 1.8.0 |
| Font | Inter (via `next/font/google`) |

**Source:** CLAUDE.md (Tailwind CSS v4 + shadcn/ui + Lucide React locked), RESEARCH.md (shadcn CLI 4.2.0, Lucide 1.8.0 verified).

**shadcn initialization command:**
```bash
npx shadcn@latest init
```

**Phase 1 components to add:**
```bash
npx shadcn@latest add button card
```

These two components are the minimum needed to verify shadcn integration works. Future phases add components as needed.

**Font rationale:** Inter is the standard font for developer tools and dashboards. It has excellent legibility at small sizes (important for terminal-adjacent UI), wide language support, and is available free via `next/font/google` with zero layout shift. shadcn/ui documentation uses Inter as its reference font.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, input padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, card body spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, page margins on desktop |
| 3xl | 64px | Page-level vertical spacing |

Exceptions: Touch targets for mobile must be minimum 44px (per WCAG 2.5.5). This applies to all interactive elements starting in Phase 2. Phase 1 has no interactive elements.

**Implementation note:** Tailwind CSS v4 uses CSS-first configuration via `@theme` directives. The spacing scale above maps to Tailwind's default spacing utilities (`p-1` = 4px, `p-2` = 8px, `p-4` = 16px, `p-6` = 24px, `p-8` = 32px, `p-12` = 48px, `p-16` = 64px). No custom spacing tokens are needed -- use Tailwind defaults.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base font-normal leading-relaxed` |
| Label | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold leading-snug` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` |
| Display | 28px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-tight` |

**Constraints:**
- Exactly 2 font weights used across the entire application: 400 (regular) for body and muted text, 600 (semibold) for labels, navigation items, headings, and display headings. No other weights are permitted.
- Body text never goes below 14px.
- Code/monospace text uses `font-mono` (system monospace stack via Tailwind) at 14px weight 400 line-height 1.6.

**Source:** Defaults aligned with shadcn/ui's typography conventions. 16px body at 1.5 line-height is the web accessibility standard.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` / `hsl(240 10% 3.9%)` (dark) | Page background, main content surfaces |
| Secondary (30%) | `hsl(240 4.8% 95.9%)` / `hsl(240 3.7% 15.9%)` (dark) | Cards, sidebar, navigation bar, input backgrounds |
| Accent (10%) | `hsl(240 5.9% 10%)` / `hsl(0 0% 98%)` (dark) | Primary buttons, active navigation indicator, focus rings, link text |
| Destructive | `hsl(0 84.2% 60.2%)` | Delete environment, remove connection -- destructive actions only |

**Accent reserved for (across all phases):**
1. Primary action buttons (e.g., "Create Environment" in Phase 3)
2. Active/selected navigation indicator (Phase 6 dashboard)
3. Focus rings on interactive elements (Phase 2 onward)
4. Inline text links

Accent is NEVER used for: decorative borders, background tints, status indicators, or informational badges.

**Status colors (defined now, used starting Phase 3):**

| Status | Color | Usage |
|--------|-------|-------|
| Running | `hsl(142 71% 45%)` (green) | Environment running indicator |
| Stopped | `hsl(240 3.8% 46.1%)` (muted gray) | Environment stopped indicator |
| Error | `hsl(0 84.2% 60.2%)` (red) | Environment error state |
| Starting | `hsl(48 96% 53%)` (amber) | Environment transitioning state |

**Implementation note:** These values are shadcn/ui's default zinc-based theme. They are defined in the CSS variables generated by `npx shadcn@latest init`. Do not override them in Phase 1. If a different theme is desired, it should be changed before Phase 2 begins by re-running `npx shadcn@latest init` with a different base color.

**Dark mode:** The application defaults to dark mode (`class="dark"` on `<html>`). Developer tools and terminal-heavy applications are conventionally dark-themed. The root layout must include dark mode support via the `dark` class strategy in Tailwind. Users will be able to toggle light/dark in a later phase if desired.

## Root Layout Contract

The Phase 1 root layout (`src/app/layout.tsx`) establishes the shell that all future pages render inside.

| Property | Value |
|----------|-------|
| HTML lang | `en` |
| Theme | Dark by default (`class="dark"` on html) |
| Font loading | Inter via `next/font/google`, applied to `<body>` |
| Body classes | `font-sans antialiased bg-background text-foreground` |
| Metadata title | "DevDock" |
| Metadata description | "Self-hosted remote development platform" |
| Viewport | `width=device-width, initial-scale=1` (Next.js default) |

**Root page (`src/app/page.tsx`):**
- Center-aligned content, vertically and horizontally centered
- Display the heading "DevDock" at display size (28px semibold)
- Display subheading "Remote development platform" at body size (16px regular), muted color
- Display body text "Dashboard coming soon. System is running." at label size (14px semibold), muted color
- No navigation, no sidebar, no interactive elements
- This page will be replaced by a redirect to `/dashboard` in Phase 2

## Accessibility Baseline

Established in Phase 1, enforced from Phase 2 onward:

| Requirement | Standard | Implementation |
|-------------|----------|----------------|
| Color contrast | WCAG AA (4.5:1 for text, 3:1 for large text) | shadcn/ui default theme meets this in both light and dark mode |
| Focus indicators | Visible focus ring on all interactive elements | shadcn/ui includes `focus-visible:ring-2` by default |
| Touch targets | Minimum 44x44px for interactive elements | Enforce starting Phase 2 |
| Semantic HTML | Use `<main>`, `<nav>`, `<header>`, `<footer>` | Root layout uses `<main>` for content area |
| Reduced motion | Respect `prefers-reduced-motion` | Tailwind's `motion-reduce:` variant available |
