# Phase 999.2 Plan 04: Container Entrypoints Summary

**One-liner:** Two PID-1 shell entrypoints implementing postgres-wait, runtime docker-socket GID fixup, drizzle-kit schema push, idempotent admin seed, and gosu privilege drop.

## What Was Built

### scripts/entrypoint-app.sh

The `app` service entrypoint runs as root at container startup and performs five ordered steps before dropping privileges:

**Step 1 — Postgres wait** (`pg_isready` polling loop)
- Env vars: `POSTGRES_HOST` (default: `postgres`), `POSTGRES_USER` (default: `devdock`), `POSTGRES_DB` (default: `devdock`), `POSTGRES_WAIT_TIMEOUT` (default: `60s`)
- Polls every 1 second until postgres responds or timeout elapses; exits 1 on timeout

**Step 2 — Docker socket GID fixup** (RESEARCH.md §3)
- Reads live socket GID via `stat -c '%g' /var/run/docker.sock`
- `DOCKER_GID` env var overrides detected GID (for macOS Docker Desktop compatibility)
- Creates group if it doesn't exist; adds `nextjs` user to that group
- No-op if `/var/run/docker.sock` is not mounted (portable across deployment modes)

**Step 3 — drizzle-kit push** ([BLOCKING] schema push requirement)
- Invokes `npx --no-install drizzle-kit push --config=drizzle.config.ts`
- Runs WITHOUT the force flag (T-999.2-06 fail-closed policy): destructive schema changes cause non-zero exit on non-TTY, aborting boot safely
- `npx --no-install` fails fast if drizzle-kit is absent from node_modules

**Step 4 — Admin seed** (idempotent)
- Invokes `npx --no-install tsx src/scripts/seed-admin-boot.ts` (Plan 02)
- Seed script itself is a no-op when `ADMIN_USERNAME`/`ADMIN_PASSWORD_HASH` are unset or users table already has rows
- Entrypoint logs whether seed vars are set — never logs the value of `ADMIN_PASSWORD_HASH` or `POSTGRES_PASSWORD`

**Step 5 — Privilege drop**
- Creates `.next/cache` dir and chowns to `nextjs:nodejs`
- `exec gosu nextjs "$@"` — drops to `nextjs` user; if gosu is missing, fails closed (no silent root execution)

### scripts/entrypoint-terminal.sh

The `terminal` service entrypoint is minimal: GID fixup (same pattern as app, with `termuser` instead of `nextjs`), then `exec gosu termuser "$@"`. No DB access, no migrations, no seeding — the terminal server has no database dependency (RESEARCH.md Open Questions #6).

## Security Properties

| Threat | Mitigation Applied |
|--------|-------------------|
| T-999.2-03 Information Disclosure | `log()` function never prints values of `ADMIN_PASSWORD_HASH`, `AUTH_SECRET`, or `POSTGRES_PASSWORD` — only presence/absence is logged |
| T-999.2-05 Elevation of Privilege | `exec gosu nextjs/termuser` — if gosu is missing, container exits; no silent root execution |
| T-999.2-06 Tampering via drizzle | No `--force` flag — destructive prompts cause non-zero exit on non-TTY, boot fails safely |
