    ## Step 1 — Prepare host directories (prevent root-owned bind-mount issues)

    DevDock's compose mirror-mounts host paths (D-09). If those directories don't exist at `docker compose up` time, Docker creates them as root, breaking the non-root nextjs user's writes (RESEARCH.md Pitfall 10).

    ```bash
    sudo mkdir -p /srv/devdock/data /srv/devdock/apps-empty /srv/devdock/claude-empty
    # If you use production apps monitoring:
    # sudo mkdir -p /home/murx/apps  (already exists in prod per project history)
    # If you use Claude Code shared config:
    # Confirm your CLAUDE_CONFIG_PATH exists on host; must match the value you set in .env.local.

    # Ownership: DevDock's app + terminal containers run as uid 1001 (nextjs / termuser).
    # For DEVDOCK_DATA_DIR the container needs write access. Two options:
    #   A) chown to uid 1001 (preferred):
    sudo chown -R 1001:1001 /srv/devdock/data
    #   B) Or chmod 0775 with the docker group: situational — ask an admin.
    ```

    ```bash
    cd /home/murx-dev/devdock

    # On your laptop, NOT the VPS:
    #   git clone ... && cd devdock && npm install && npm run hash-password -- 'your-strong-password'
    # Copy the resulting hash into .env.local's ADMIN_PASSWORD_HASH line.

    # Populate .env.local with:
    #   POSTGRES_PASSWORD=<strong random>
    #   AUTH_SECRET=<openssl rand -base64 32>
    #   ADMIN_USERNAME=admin
    #   ADMIN_PASSWORD_HASH=$2b$12$...        (pasted from local machine)
    #   DEVDOCK_DATA_DIR=/srv/devdock/data
    #   PRODUCTION_APPS_DIR=/home/murx         (or your actual apps dir)

    #   PREVIEW_DOMAIN=preview.devdock.yourdomain.com
    #   NEXT_PUBLIC_PREVIEW_DOMAIN=preview.devdock.yourdomain.com

    #   DATABASE_URL=postgresql://devdock:<POSTGRES_PASSWORD>@postgres:5432/devdock
    #   AUTH_URL=https://devdock.yourdomain.com

    # Verify file perms — .env.local must NOT be world-readable:

    ```
