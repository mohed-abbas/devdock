## Key Design Decisions

### D-17 Single Dockerfile, Two Targets

Plan 03 uses a single `Dockerfile` with `--target app-runner` and `--target terminal-runner` instead of a separate `Dockerfile.terminal`. The decision rationale: both services share the same `deps` stage (`npm ci` on the same `package.json`), so a single file avoids duplicating the deps install and naturally expresses the build dependency graph. This is D-17 "Claude's discretion."

### Full node_modules COPY (Not Selective drizzle-kit Copy)

The earlier research draft discussed copying only `/node_modules/drizzle-kit` into the app-runner. This was rejected because drizzle-kit's transitive runtime dependencies — `esbuild`, `jiti`, `@drizzle-team/brocli`, `hanji` — are not auto-included. Copying the entire `deps`-stage `node_modules` is the gap-closure fix:

```
COPY --from=deps --chown=nextjs:nodejs /app/node_modules ./node_modules
```

This line is grep-verified in acceptance criteria and is the key correctness guarantee for the BLOCKING `drizzle-kit push` step in Plan 04's entrypoint.

### No Stub Entrypoint Scripts

The Dockerfile contains:
```
COPY --chmod=0755 scripts/entrypoint-app.sh /entrypoint.sh
COPY --chmod=0755 scripts/entrypoint-terminal.sh /entrypoint.sh
```

These reference scripts owned by Plan 04 (`files_modified: [scripts/entrypoint-*.sh]`). Plan 03 does NOT create stub versions of these files. The `docker build` that uses this Dockerfile will fail until Plan 04 lands — this is expected and by design (Plan 10 runs the first full build, which runs after Plan 04 is complete).

### No Image Build in This Plan

All acceptance criteria are text/grep-only on the Dockerfile file content. No `docker build` was attempted. Build verification is Plan 10's responsibility (Wave 7, after Plan 04 ships real entrypoints).

## Caddyfile Configuration

- **Admin API:** `admin :2019` — binds to all interfaces on compose-internal network only; NEVER published to host (T-999.2-02)
- **TLS:** `auto_https off` — HTTP-only inside the stack; host nginx terminates TLS (D-03)
- **Persistence:** `persist_config on` — dynamic preview routes (added via Admin API) survive restarts
- **Routing:**
  - `handle /ws/*` → `reverse_proxy terminal:3001` (Socket.IO path per `server/terminal-server.ts` line 17)
  - `handle` (catch-all) → `reverse_proxy app:3000` (Next.js)
