# Phase 999.2 Plan 06: Caddy Admin API Client Summary

Typed Caddy Admin API client with idempotent route add/remove, dynamic server key discovery, and input validation via allow-list regexes. All 11 unit tests pass with mocked fetch.

## What Was Built

### Exported API Surface

**`src/lib/docker/caddy-admin.ts`** exports:

| Export | Signature | Purpose |
|--------|-----------|---------|
| `addPreviewRoute` | `(input: AddPreviewRouteInput) => Promise<void>` | Idempotent upsert: DELETE-then-POST to Caddy routes array |
| `removePreviewRoute` | `(envSlug: string) => Promise<void>` | DELETE by @id; 404 treated as success |
| `getServerKey` | `() => Promise<string>` | Discovers `srv0` (or first key) from `/config/apps/http/servers/`; caches result |
| `__resetServerKeyCacheForTests` | `() => void` | Test utility to reset module-level cache |

**`src/lib/docker/caddy-admin.types.ts`** exports:

- `CaddyRoute` — route JSON shape with `@id`, `match`, `handle`, `terminal`
- `AddPreviewRouteInput` — typed parameters for `addPreviewRoute`
- `CaddyAdminError` — error interface with optional `status` field

### Security Invariants (Input Validation Allow-Lists)

Per threat register T-999.2-02 (Tampering disposition: mitigate):

| Field | Constraint | Regex/Check |
|-------|-----------|-------------|
| `envSlug` | URL-safe slug | `/^[a-z0-9-]+$/` |
| `envId` | UUID format | `/^[0-9a-f]{8}-[0-9a-f]{4}-...-[0-9a-f]{12}$/i` |
| `containerName` | DevDock container naming | `/^devdock-[a-z0-9-]+-dev$/` |
| `previewPort` | Valid TCP port | Integer, 1..65535 |
| `previewDomain` | Domain/host chars | `/^[a-z0-9.\-:]+$/i` |

All validations run before any URL interpolation, preventing path-injection and route-hijack attacks.

### Idempotency Patterns

- **addPreviewRoute**: DELETE `@id` tag first (swallows errors), then POST — safe to call multiple times without duplicate routes
- **removePreviewRoute**: 404 response treated as success — safe to call when route doesn't exist

### Error Handling

- All error messages truncated to 500 characters (matches `docker-service.ts` convention: `.slice(0, 500)`)
- `getServerKey` errors include HTTP status code for diagnostics
- `addPreviewRoute` and `removePreviewRoute` surface POST/DELETE failure status codes
