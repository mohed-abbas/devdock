## Network Topology

```
host nginx (ports 80 and 443, TLS)
  └→ 127.0.0.1:${CADDY_INTERNAL_PORT:-8080}   ← loopback-only bind (D-02)
      └→ caddy (devdock-net + devdock-proxy)
          ├→ app:3000          (Next.js standalone)
          ├→ terminal:3001     (Socket.IO /ws/*)
          └→ devdock-{slug}-dev:{port}  (preview containers on devdock-proxy)
```

### Networks

| Network | Name | Type | Purpose |
|---------|------|------|---------|
| `devdock-net` | devdock-net | bridge (stack-owned) | Internal DevDock service communication |
| `devdock-proxy` | devdock-proxy | bridge (stack-owned, external: false) | Preview routing — Caddy reaches per-project dev containers here; per-project composes reference it with `external: true` |

Caddy attaches to BOTH networks — this is the [BLOCKING] §5 requirement from RESEARCH.md.

Subnet assignment: `devdock-proxy` uses `172.30.0.0/24` to avoid collisions with per-project networks.

## Service Startup Ordering

```
postgres (healthcheck: pg_isready)
  └→ caddy (healthcheck: wget :2019/config/)
      └→ app (condition: service_healthy on both postgres + caddy)
          └→ terminal (condition: service_started on app)
```

## Security Invariants Enforced by Structure

| Threat ID | Invariant | Enforced By |
|-----------|-----------|-------------|
| T-999.2-01 | Docker socket in app + terminal ONLY — never in caddy or postgres | Caddy/postgres services have no socket volume entry |
| T-999.2-02 | Caddy port 2019 NEVER in `ports:` | No `2019:2019` mapping; caddy `ports:` only has `:8080` loopback bind |

## docker-compose.override.yml

Auto-loaded by `docker compose up` (compose v2 convention). Overlays dev settings on top of production compose:

- `app` and `terminal` services switch to `target: deps` build stage (has all devDependencies)
- `app` CMD overridden to `npx next dev --turbopack --port 3000 --hostname 0.0.0.0`
- `terminal` CMD overridden to `npx tsx --watch server/terminal-server.ts`
- Source bind-mounts for HMR: `./src`, `./server`, `./public`, config files
- Anonymous volumes `- /app/node_modules` and `- /app/.next` prevent host-arch node_modules from shadowing image's Linux-built copies
- `NODE_ENV: development` for both services
- `entrypoint: []` disables production entrypoint (skips drizzle-push + seed for fast dev boot)
- postgres and caddy are NOT overridden — identical in dev and prod

**To run production stack without dev overrides:**
```bash
docker compose -f docker-compose.yml up -d --build
```

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Created gitignored `.env` for compose-lint.sh execution**
- **Found during:** Pre-Task 2 verification
- **Issue:** `docker compose config` fails loudly when `POSTGRES_PASSWORD` and `AUTH_SECRET` are absent (`:?` fail-closed syntax from the plan's compose content). `.env.local` exists for Next.js dev but Docker Compose does not load `.env.local` — it only reads `.env`. Without a `.env` file, `tests/compose-lint.sh` exits non-zero.
- **Fix:** Created a gitignored `.env` file with placeholder values (`POSTGRES_PASSWORD=compose-lint-test-placeholder`, `AUTH_SECRET=compose-lint-test-placeholder-32chars-min`). File is not committed (.env is in .gitignore).
- **Files modified:** `.env` (untracked/gitignored — not in any commit)
- **Commit:** N/A (not committed — gitignored)
