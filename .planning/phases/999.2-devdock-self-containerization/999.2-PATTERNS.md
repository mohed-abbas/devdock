### `Dockerfile` (repo root) ŌĆö multi-stage Next.js app container

**Analog:** `docker/base/Dockerfile` (weak ŌĆö Ubuntu single-stage user-project base; only specific idioms transfer)

**Transferable idioms ŌĆö ARG declarations** (`docker/base/Dockerfile` lines 7-9):
```dockerfile
ARG HOST_UID=1000
ARG HOST_GID=1000
ARG NODE_MAJOR=22
```
Copy this ARG-with-default pattern. For the app Dockerfile, use `ARG DOCKER_GID=999` and `ARG NEXT_PUBLIC_PREVIEW_DOMAIN` (build-time only).

**Transferable idiom ŌĆö non-root user creation with conflict guard** (`docker/base/Dockerfile` lines 38-50):
```dockerfile
RUN if getent group ${HOST_GID} >/dev/null 2>&1; then \
      groupmod -n dev $(getent group ${HOST_GID} | cut -d: -f1); \
    else \
      groupadd -g ${HOST_GID} dev; \
    fi \
    && if getent passwd ${HOST_UID} >/dev/null 2>&1; then \
      existing_user=$(getent passwd ${HOST_UID} | cut -d: -f1); \
      usermod -l dev -d /home/dev -m -g dev -s /bin/bash "$existing_user"; \
    else \
      useradd -m -u ${HOST_UID} -g dev -s /bin/bash dev; \
    fi \
    && echo "dev ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers.d/dev
```
The app Dockerfile uses a simpler static user (`nextjs:nodejs`) since GID fixup happens at runtime in the entrypoint.

**Transferable idiom ŌĆö entrypoint copy + chmod + ENTRYPOINT exec form** (`docker/base/Dockerfile` lines 52-55, 65):
```dockerfile
COPY entrypoint.sh /usr/local/bin/entrypoint.sh
RUN chmod +x /usr/local/bin/entrypoint.sh
...
ENTRYPOINT ["/usr/local/bin/entrypoint.sh"]
```

**Multi-stage structure and Next.js standalone COPY pattern:** No analog in repo. Use RESEARCH.md ┬¦1 (Implementation Patterns, pattern 1: `deps ŌåÆ builder ŌåÆ runner`, three explicit `COPY --from=builder` lines for `public/`, `.next/standalone/`, `.next/static/`, and the drizzle-kit copy pattern). Note: `node:22-slim` (Debian) is required ŌĆö not Alpine ŌĆö due to bcrypt native addon (RESEARCH.md, bcrypt section).
