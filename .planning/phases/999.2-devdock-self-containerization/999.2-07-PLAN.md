<objective>
Update the per-project compose template so user-project dev containers attach to the shared `devdock-proxy` network. This is the other half of the contract Plan 05 established: the root stack OWNS `devdock-proxy` (external: false, name: devdock-proxy); every generated per-project compose REFERENCES it with `external: true, name: devdock-proxy` and attaches the `dev` service. Caddy (also on `devdock-proxy`) can then reach `devdock-{slug}-dev:{port}` for preview routing.

Cross-reference (informational): D-10 — shared external Docker network devdock-proxy.

Purpose: Lock the network contract that Phase 999.1 (project-native compose injection) builds on.
Output: Template update + generator update (minimal — template does the heavy lifting) + regression test.
</objective>
