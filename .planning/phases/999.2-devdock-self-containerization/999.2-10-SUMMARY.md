### ❌ entrypoint.test.sh (FAIL — GAP-1)

```
[2026-04-19T00:26:48+02:00] entrypoint.test: checking scripts/entrypoint-app.sh
[2026-04-19T00:26:50+02:00] entrypoint.test: first run
[2026-04-19T00:26:50+02:00] entrypoint-app: starting (pid 72364)
[2026-04-19T00:26:50+02:00] entrypoint-app: waiting for postgres at postgres (timeout=60s)
[2026-04-19T00:27:51+02:00] entrypoint-app: postgres not ready after 60s — giving up
exit code 1
```

**Root cause:** `scripts/entrypoint-app.sh` defaults `POSTGRES_HOST=postgres` and never reads `POSTGRES_PORT`. The test runs the entrypoint on the host against a throwaway postgres bound to `127.0.0.1:55432` but only exports `DATABASE_URL` (which the wait loop ignores) — `pg_isready -h postgres` resolves to nothing on the host.

**Fix scope (gap-closure 999.2.1):**
- Patch `scripts/entrypoint-app.sh`: pass `-p "${POSTGRES_PORT:-5432}"` to `pg_isready`
- Patch `tests/entrypoint.test.sh`: `export POSTGRES_HOST=127.0.0.1; export POSTGRES_PORT=55432` before invoking the entrypoint
