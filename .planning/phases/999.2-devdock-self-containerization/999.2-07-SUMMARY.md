## Decisions Made

- **No generator code change required:** The `devdock-proxy` network lines are static YAML with no `{{PLACEHOLDER}}` markers, so they pass through `generateComposeFile`'s template substitution pipeline verbatim. The `uncommentSection`/`uncommentVolumes` helpers only act on patterns matching `# postgres:`, `# redis:`, and root-level `# volumes:` — the new `devdock-proxy` block matches none of these.
- **Real template in regression test:** The new test uses `vi.importActual` to read the actual `docker/templates/base-compose.yml` from disk rather than the inline `TEMPLATE_CONTENT` constant, so any future edit to the template that removes `devdock-proxy` will immediately break the test. The inline constant is not updated — it is stable for all other existing tests.
