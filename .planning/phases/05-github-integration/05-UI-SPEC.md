# Phase 5 -- UI Design Contract

> Visual and interaction contract for the GitHub integration phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 5 delivers three UI areas:** a Settings page with GitHub OAuth connect/disconnect, an enhanced creation dialog with searchable repo dropdown and branch selector, and supporting empty/error states for GitHub connectivity. The existing creation dialog is augmented -- not replaced. No dashboard polish (Phase 6), no resource limits (Phase 7).

---

## Enhanced Creation Dialog (D-04, D-05, D-06, D-07, D-08)

### Structure Changes

The existing creation dialog is modified. When GitHub is connected, the "Git Repository URL" manual input is replaced by a searchable repo combobox. When not connected, the manual URL input remains with a hint.

```
Dialog (inherited, max-w-md)
|
+-- DialogHeader (unchanged)
|
+-- Form (space-y-4)
    |
    +-- Field: Name (unchanged, required)
    |
    +-- IF GITHUB CONNECTED:
    |   +-- Field: Repository (D-04)
    |   |   +-- Label "Repository"
    |   |   +-- Combobox (Popover + Command)
    |   |   |   +-- Trigger: "Select a repository..." placeholder
    |   |   |   +-- Dropdown:
    |   |   |       +-- CommandInput: search/filter field
    |   |   |       +-- CommandList (max-h-64, overflow-y-auto)
    |   |   |       |   +-- CommandEmpty: "No repositories found."
    |   |   |       |   +-- CommandGroup:
    |   |   |       |       +-- CommandItem (repeated):
    |   |   |       |           +-- owner/repo text
    |   |   |       |           +-- Lock icon (if private)
    |   |   |       |           +-- Check icon (if selected)
    |   |   +-- Link: "Enter URL manually" (text-sm, underline, muted-foreground)
    |   |
    |   +-- IF REPO SELECTED:
    |       +-- Field: Branch (D-08)
    |           +-- Label "Branch"
    |           +-- Select dropdown (shadcn Select)
    |               +-- Default branch pre-selected
    |               +-- Other branches listed alphabetically
    |
    +-- IF GITHUB NOT CONNECTED:
    |   +-- Field: Git Repository URL (unchanged)
    |   +-- Hint: "Connect GitHub in Settings to browse your repos." (text-sm, muted-foreground, mt-1)
    |
    +-- IF "Enter URL manually" clicked:
    |   +-- Field: Git Repository URL (same as not-connected)
    |   +-- Link: "Select from GitHub" (text-sm, underline, muted-foreground)
    |
    +-- Separator (unchanged)
    +-- Sidecar services (unchanged)
    +-- Error region (unchanged)
    +-- DialogFooter (unchanged)
```

### Combobox Specifications (D-05, D-06)

| Property | Value |
|----------|-------|
| Implementation | shadcn Popover + Command pattern |
| Trigger | Button variant="outline", `w-full justify-between`, `text-sm` |
| Trigger text (empty) | "Select a repository..." in muted-foreground |
| Trigger text (selected) | `owner/repo` in foreground |
| Trigger icon | `ChevronsUpDown` from lucide, `size-4 text-muted-foreground`, right side |
| Dropdown width | Same as trigger (use `popoverContentProps={{ style: { width: 'var(--popover-trigger-width)' } }}`) |
| Search input placeholder | "Search repositories..." |
| Item format | `owner/repo` text, left-aligned |
| Private repo indicator | `Lock` icon from lucide, `size-3 text-muted-foreground`, after repo name, `ml-1` |
| Selected indicator | `Check` icon from lucide, `size-4`, right side of item |
| Sort order | Most recently pushed first (D-06) |
| Empty results | "No repositories found." in `text-sm text-muted-foreground`, centered |
| Loading state | "Loading repositories..." with spinner icon in dropdown body |
| Max visible items | ~7 items visible, scrollable for more |
| Filter | Client-side filter as user types (D-05) |
| Cache | ~5 minute cache on fetched repo list (D-05) |

| Property | Value |

## Interaction States

### Settings Page States

| State | Visual Treatment |
|-------|-----------------|
| **GitHub not connected** | Card shows "Connect GitHub" button. No avatar, no username. |
| **OAuth in progress** | "Connect GitHub" button shows "Connecting..." with disabled state. User is redirected to GitHub OAuth page. |
| **GitHub connected** | Card shows avatar + username + "Connected" badge + "Disconnect" button. |
| **Disconnect in progress** | AlertDialog "Disconnect" button shows "Disconnecting..." with disabled state. |
| **OAuth error** | Redirect back to settings with error toast: "Failed to connect GitHub. Please try again." |
| **Token expired/revoked** | On next API call that fails with 401 from GitHub, show toast: "GitHub connection expired. Please reconnect." and reset to not-connected state. |

### Enhanced Creation Dialog States

| State | Visual Treatment |
|-------|-----------------|
| **GitHub connected, no repo selected** | Combobox trigger shows "Select a repository..." placeholder. Branch selector hidden. |
| **Repos loading** | Combobox dropdown shows "Loading repositories..." with spinner. |
| **Repos loaded, searching** | User types in search input, list filters client-side. |
| **Repo selected** | Combobox trigger shows `owner/repo`. Branch selector appears with default branch pre-selected. |
| **Branches loading** | Branch selector shows "Loading branches..." as placeholder. |
| **Branch selected** | Branch selector shows selected branch name. |
| **Manual URL mode** | Combobox hidden, manual URL input shown with "Select from GitHub" link. |
| **GitHub not connected** | Manual URL input shown with "Connect GitHub in Settings to browse your repos." hint. |
| **Repo fetch error** | Combobox dropdown shows "Failed to load repositories." with a "Retry" button. |

### Loading States

| Scenario | Visual |
|----------|--------|
| **Settings page initial load** | Skeleton card for GitHub section (shimmer, matches card dimensions). |
| **Repo list loading** | Spinner inside combobox dropdown, "Loading repositories..." text. |
| **Branch list loading** | Select trigger shows "Loading branches..." in muted text. |
| **OAuth redirect** | Button shows "Connecting..." text, disabled. Browser navigates to GitHub. |

| Success: GitHub connected | "GitHub connected." (toast) |
| Success: GitHub disconnected | "GitHub disconnected." (toast) |

**Copywriting rules (inherited from Phase 2-3):**
- No exclamation marks. Tone is calm and professional.
- Error messages use period at end. They describe what happened, not what the user did wrong.
- Button labels use title case for multi-word.
- Destructive confirmation explains consequences before asking to proceed.
- No generic "Cancel" labels. Dismiss buttons describe the consequence of not proceeding (e.g. "Keep Connected").
- Toast messages are short, one sentence, end with period.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Combobox keyboard navigation | Arrow keys navigate items, Enter selects, Escape closes dropdown. Built into Command component. |
| Combobox ARIA | Popover trigger has `role="combobox"`, `aria-expanded`, `aria-controls`. Command items have `role="option"`. |
| Select keyboard navigation | Arrow keys navigate, Enter selects, Escape closes. Built into Select component. |
| Form labels | Every input, combobox, and select has a `<Label>` with matching `htmlFor`/`id`. |
| Private repo indicator | Lock icon has `aria-label="Private repository"` for screen readers. |
| Avatar alt text | `alt="{username}'s GitHub avatar"` |
| Toast announcements | Toast messages use `aria-live="polite"` (built into toast component). |
| Focus management | After OAuth redirect back to settings, focus returns to the GitHub card. |
| Destructive action guard | Disconnect requires explicit AlertDialog confirmation. |
| Touch targets | All buttons minimum 44px. Combobox items have sufficient padding for touch. |
