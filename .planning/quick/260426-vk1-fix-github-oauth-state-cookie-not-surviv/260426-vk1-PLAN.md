Two file changes, one atomic commit.

**Change 1 — `src/app/api/github/authorize/route.ts` line 18:**
Replace:
```ts
    secure: true,
```
with:
```ts
    secure: config.AUTH_URL?.startsWith('https://') ?? false,
```
Rationale: when AUTH_URL is `https://...` (production behind Caddy), the cookie keeps the Secure attribute and browsers send it back over the encrypted callback. When AUTH_URL is `http://localhost:8080` (dev) or unset, the Secure attribute is dropped so the browser will resend the cookie on the HTTP callback — fixing the `state_mismatch` failure path. `config` is already imported at line 4; no new imports needed. The remaining flags (`httpOnly: true`, `sameSite: 'lax'`, `maxAge: 600`, `path: '/'`) stay intact — those are sufficient CSRF/lifetime protections for a 10-minute OAuth state cookie.

**Change 2 — `docker-compose.yml`:**
Do NOT modify; the change is already in the working tree (verify with `git diff docker-compose.yml`). Just stage it as part of this commit. The diff adds a comment block plus three bare-interpolation env entries (`GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET`, `GITHUB_TOKEN_ENCRYPTION_KEY`) on the `app` service env block between `ADMIN_PASSWORD_HASH` and `CADDY_ADMIN_URL`. Bare interpolation (no `:-` default) is intentional per the inline comment: an unset var produces a compose warning and an empty string that trips zod at boot — louder than the previous "GitHub integration not configured" silent-fail mode.
