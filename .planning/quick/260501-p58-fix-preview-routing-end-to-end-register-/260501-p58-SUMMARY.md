# Quick Task 260501-p58: Fix preview routing end-to-end

**One-liner:** Six independent bugs in the preview-routing chain. None of them individually blocking, but together they meant preview never worked end-to-end. Fixed all six and validated via curl on the real nip.io domain.

## What Was Done

The original plan named three bugs. While verifying, three more surfaced — each one shadowed by the previous in the chain. Listed in fix order:

| # | Layer | Bug | Fix |
|---|-------|-----|-----|
| 1 | API | PATCH didn't register Caddy route on previewPort change | Wire `registerPreviewRoute` / `deregisterPreviewRoute` into PATCH handler for running envs |
| 2 | Config | `PREVIEW_DOMAIN=…:3000` (legacy port) but Caddy publishes :8080 | Update `.env*` to `…nip.io:8080` |
| 3 | Caddy | Auth canonical-host redirect (260501-ihv) caught preview subdomains | Add second `not` clause to `@noncanonical` matcher to exempt `*.PREVIEW_HOST` |
| 4 | Caddy | Admin API rejected app's fetches with 403 "origin ''" | Caddyfile `origins` allowlist + `Origin` header pinned on every admin fetch (undici sends `sec-fetch-mode:cors` without setting Origin) |
| 5 | Caddy | Preview routes appended to end-of-array, AFTER catch-all to app:3000 | Switch `POST /routes` (append) → `PUT /routes/0` (head insert); `terminal:true` keeps it safe |
| 6 | Compose | `NEXT_PUBLIC_PREVIEW_DOMAIN` only set as build arg, not runtime env | Add to `app` service `environment:` block so dev mode (Turbopack) sees it |
